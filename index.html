Three presentation-ready reports comparing recurrent gating and long-context attention on thermal failure, cloud telemetry, and cross-domain demand data

Model Architectures

The repository compares compact model families under a common evaluation discipline: time-respecting splits, forecast error, incident sensitivity, ranking quality, and visualization of failure modes.

RNN

VanillaRNN

Minimal recurrent baseline used to test whether adaptive gating is genuinely necessary. In the real failure notebook it is decisively worse than the gated alternatives on both forecast error and ranking.

  • Role: weak recurrent baseline
  • Best shown in RNN report
  • Parameters: 5,645
LSTM

LSTMForecaster

High-capacity gated recurrent reference. It is the strongest mean-F1 model in the cross-dataset benchmark and also wins MAE on the EC2 CPU dataset by a narrow margin.

  • Best aggregate mean F1: 0.1057
  • EC2 CPU MAE winner
  • Parameters: 116,845
GRU

GRUForecaster

Lighter gated recurrent model with the best overall accuracy-efficiency profile in this repository. It achieves the lowest mean MAE in the combined benchmark and the best incident-aware balance in the RNN report.

  • Lowest mean MAE: 1337.33
  • Best mean ROC-AUC: 0.6603
  • Parameters: 87,949
Transformer

TransformerForecaster

Attention-based long-context model used where periodic alignment and calibrated ranking matter more than a single binary threshold. It is most convincing in the cloud telemetry notebook, where visual evidence is stronger than the raw test-set F1.

  • EC2 CPU ROC-AUC: 0.7987
  • Threshold selected: 0.1
  • Parameters: 226,765
AE

AnomalyDetector

Reconstruction-based detector paired with the Transformer report to test whether anomaly scores concentrate in incident windows rather than simply following variance spikes.

  • Used in Transformer report
  • Best shown via score histograms
  • Supports threshold analysis

Best Visual Evidence

The most useful way to present this repository is through the plots that isolate why one model family wins over another.

01

Residual Separation

The recurrent report's residual boxplots show that incident windows are not merely harder forecasts; they form a separable error regime once adaptive memory is introduced.

02

Threshold Sensitivity

The Transformer report makes threshold dependence explicit rather than hiding it. This is the right way to discuss anomaly detection when ROC-AUC is strong but the chosen test threshold does not maximize F1.

03

Cross-Dataset Frontier

The capstone frontier plot summarizes the trade-off that matters most: low forecast error, useful event sensitivity, and credible ranking quality do not collapse to a single winner on every dataset.

04

Leader Margins

The win-count and runner-up margin panels prevent overclaiming. They show where `GRU` wins decisively, where `LSTM` wins narrowly, and where the right model depends on the metric being optimized.

Quick Start

Reproduce the notebook environment and regenerate the executed reports.

# 1. Clone & install
git clone https://github.com/mohuyn/Quantum-Drift-Forecasting.git
cd Quantum-Drift-Forecasting
conda activate qaoa
pip install -r requirements.txt

# 2. Re-run the three reports
jupyter nbconvert --to notebook --execute --inplace notebooks/rnn_drift_forecast.ipynb
jupyter nbconvert --to notebook --execute --inplace notebooks/transformer_calibration.ipynb
jupyter nbconvert --to notebook --execute --inplace notebooks/quantum_drift_combined.ipynb

# 3. Export presentation-ready HTML
jupyter nbconvert --to html --output-dir website/notebooks_html \
  notebooks/rnn_drift_forecast.ipynb \
  notebooks/transformer_calibration.ipynb \
  notebooks/quantum_drift_combined.ipynb

# 4. Start the optional inference API
python -m src.server --port 5000

# 5. Open the landing page
open index.html
