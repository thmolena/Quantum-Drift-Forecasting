Adaptive Sequence Modeling For Real Operational Time Series

A compact benchmark of recurrent gating and long-context attention on thermal failure, cloud telemetry, and cross-domain demand data.

The repository is strongest when read as an objective-aware benchmark rather than a claim of universal architectural superiority. In this study, GRU is the most robust low-error aggregate choice, LSTM is the strongest mean-F1 model across datasets, and the Transformer report is most convincing as a calibration and ranking-quality result.

Key Findings

The empirical argument fits into three claims, each tied to a specific report and selection objective.

01

GRU Improves Incident-Aware Forecasting

On the thermal failure trace, GRU achieved MAE 51.79, RMSE 55.35, F1 0.257, and ROC-AUC 0.718, outperforming the plain recurrent baseline while remaining lightweight enough for CPU-oriented reproduction.

02

Transformer Is Strongest As A Calibration Result

On EC2 CPU telemetry, the Transformer reached MAE 0.0436 and ROC-AUC 0.7987. The strongest interpretation comes from threshold response and anomaly ranking, not from a single-threshold F1 headline.

03

Cross-Dataset Selection Is Metric-Dependent

In the combined benchmark, GRU produced the best mean MAE and mean ROC-AUC, while LSTM delivered the best mean F1. The correct conclusion is therefore objective-aware model choice.

Model Families

The architecture set is intentionally compact so the comparison stays interpretable.

RNN

VanillaRNN

Weak recurrent baseline used to test whether adaptive memory is genuinely necessary on the failure-bearing telemetry trace.

LSTM

LSTMForecaster

Highest-capacity gated recurrent baseline and the strongest mean-F1 model in the combined benchmark.

GRU

GRUForecaster

Best aggregate low-error model in this repository and the clearest practical winner in the recurrent benchmark.

Transformer

TransformerForecaster

Long-context attention model used for periodic alignment, calibration analysis, and anomaly ranking on cloud telemetry.

Reproduce

Use the qaoa conda environment for notebooks, exports, and the optional local API. 

conda activate qaoa
pip install -r requirements.txt

jupyter nbconvert --to notebook --execute --inplace notebooks/rnn_drift_forecast.ipynb
jupyter nbconvert --to notebook --execute --inplace notebooks/transformer_calibration.ipynb
jupyter nbconvert --to notebook --execute --inplace notebooks/quantum_drift_combined.ipynb

jupyter nbconvert --to html --output-dir website/notebooks_html \
  notebooks/rnn_drift_forecast.ipynb \
  notebooks/transformer_calibration.ipynb \
  notebooks/quantum_drift_combined.ipynb

python -m src.server --port 5000
